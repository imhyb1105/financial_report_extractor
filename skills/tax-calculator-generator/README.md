## 1. 概述

### 1.1 什么是 Tax Calculator Generator？

Tax Calculator Generator（税种计算器生成器）是一个 **Claude Skill**，用于自动为中国各类税种生成完整的网页计算器套件。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **零中断执行** | 自动完成所有步骤，不询问用户确认 |
| **智能复杂度判断** | 自动识别税种复杂度，选择合适的模板 |
| **联网搜索税率** | 自动搜索最新税率政策 |
| **知识库生成** | 生成完整的税法知识库文档 |
| **自动测试** | 生成后自动进行语法和逻辑测试 |
| **统一性规范** | 所有生成的计算器遵循统一的风格和规范 |

### 1.3 输出内容

每个计算器包含以下完整套件：

```
{税种名称}计算器/
├── PRD.md                        # 产品需求文档
├── {税种名称}计算器.html          # 可交互的网页计算器
├── script.js                     # 计算逻辑 JavaScript
├── styles.css                    # 样式文件
├── {tax_type}.config.json        # 税率配置文件
└── {税种名称}知识库/              # 税法知识库
    ├── 0_索引.md
    ├── 1_{税种名称}基础知识.md
    ├── 2_{税种名称}税率.md
    ├── 3_{税种名称}规定.md
    ├── 4_{税种名称}计算方法.md
    └── 5_{税种名称}纳税调整详解.md   # 复杂税种专用
```

## 2. Skill 构建过程

### 2.1 设计背景

企业所得税计算的核心难点是**应纳税所得额的计算**，而非简单的税率应用：

```
应纳税所得额 ≠ 简单的收入 - 成本

实际计算流程：
应纳税所得额 = 会计利润总额
            + 纳税调整增加额（限额扣除超限、不得扣除项目）
            - 纳税调整减少额（免税收入、加计扣除、亏损弥补）
```

### 2.2 迭代演进

| 版本 | 解决的问题 | 核心改进 |
|------|-----------|----------|
| v1.0 | 初始版本 | 生成基本计算器 |
| v2.0 | 复杂度分层 | 增加简单/中等/复杂税种判断 |
| v3.0 | 应纳税所得额计算 | 增加分步向导模式，自动计算限额扣除 |
| v3.1 | 统一性规范 | 集成统一性说明文档，确保风格一致 |

### 2.3 构建原则

1. **静默执行协议**：禁止中断询问，一次性完成所有文件
2. **智能模板选择**：根据税种复杂度自动选择合适的模板
3. **知识库合规**：确保引用正确的法律法规
4. **自动测试修复**：生成后自动测试，发现问题自动修复
5. **统一性规范**：所有生成的计算器遵循统一的风格和规范

## 3. 核心功能与作用

### 3.1 主要功能

#### 功能一：税种复杂度自动判断

```
简单税种：印花税、契税、附加税、车船税、车辆购置税、关税
→ 生成单页简单表单

中等税种：房产税、土地增值税、消费税、城镇土地使用税、资源税
→ 生成单页增强表单（Tab切换）

复杂税种：企业所得税、个人所得税、增值税
→ 生成分步向导模式（5步骤）
```

#### 功能二：限额扣除自动计算（企业所得税）

| 项目 | 扣除标准 | 超限处理 |
|------|----------|----------|
| 业务招待费 | Min(实际60%, 收入5‰) | 调增，不结转 |
| 广告费 | 收入15% | 调增，可结转 |
| 职工福利费 | 工资总额14% | 调增，不结转 |
| 工会经费 | 工资总额2% | 调增，不结转 |
| 职工教育经费 | 工资总额8% | 调增，可结转 |
| 公益性捐赠 | 利润总额12% | 调增，可结转3年 |

#### 功能三：知识库自动生成

自动生成包含法律法规、税率表、计算公式的完整知识库。

### 3.2 核心作用

1. **提高效率**：从手动编码数小时缩短到自动生成数分钟
2. **保证准确性**：自动搜索最新税率，减少人工错误
3. **标准化输出**：所有计算器遵循统一的结构和风格
4. **知识沉淀**：每个计算器配套完整的税法知识库
5. **维护便捷**：统一规范降低后续维护成本

```
"生成 + {税种名称} + 计算器"
"创建 + {税种名称} + 计算器"
"制作 + {税种名称} + 计算器"
```

### 4.2 使用示例

```
用户：生成印花税计算器

执行过程：
1. 解析税种：印花税
2. 判断复杂度：simple
3. 搜索最新印花税税率
4. 生成简单单页计算器
```

#### 示例二：生成复杂税种计算器

```
用户：生成企业所得税计算器

执行过程：
1. 解析税种：企业所得税
2. 判断复杂度：complex
3. 搜索税率、限额扣除、加计扣除等政策
```

## 6. 税种复杂度分层

### 6.1 复杂度判定规则

| 复杂度 | 税种 | 特征 | 计算器模式 |
|--------|------|------|-----------|
| **简单** | 印花税、契税、附加税、车船税、车辆购置税、关税 | 税基明确、税率固定、无复杂调整 | 单页简单表单 |
| **中等** | 房产税、土地增值税、消费税、城镇土地使用税、资源税、环境保护税 | 多档税率、需判断适用情形 | 单页增强表单（Tab切换） |
| **复杂** | **企业所得税**、**个人所得税**、增值税 | 应纳税所得额计算复杂、多项调整、分档累进 | **分步向导模式** |

### 6.2 复杂税种核心难点

#### 企业所得税

```
应纳税所得额 = 会计利润总额
            + 限额扣除超限调增（业务招待费、广告费等）
            + 不得扣除项目调增（罚款、滞纳金等）
            - 免税/不征税收入调减（国债利息、股息红利等）
            - 加计扣除（研发费用100%、残疾人工资100%）
            - 亏损弥补
```

#### 个人所得税

```
应纳税所得额 = 收入合计（工资+劳务+稿酬+特许权使用费）
            - 减除费用（60000元/年）
            - 专项扣除（社保公积金）
            - 专项附加扣除（子女教育、住房贷款等）
            - 其他扣除

应纳税额 = 应纳税所得额 × 适用税率 - 速算扣除数
```

#### 增值税

```
应纳税额 = 销项税额 - （进项税额 - 进项转出）
```

## 9. 常见问题解答

### Q1：如何判断税种复杂度？

```
简单判断规则：
- 只有单一税率或简单分档 → 简单
- 需要计算应纳税所得额 → 复杂
- 涉及进项抵扣 → 复杂
- 涉及累进税率 + 多项扣除 → 复杂
```

### Q2：企业所得税为什么需要分步向导？

因为应纳税所得额的计算涉及多个步骤：
1. 限额扣除项目（需判断是否超限）
2. 不得扣除项目（需全额调增）
3. 免税/不征税收入（需调减）
4. 加计扣除（需额外计算）
5. 亏损弥补（需查询历史）

单页表单无法清晰展示这些调整过程。

### Q3：如何处理小微企业300万临界点？

```javascript
if (taxableIncome <= 3000000) {
    // 享受小微企业优惠（5%实际税负）
} else {
    // 全部所得按25%计算（不是超限部分！）
}
```

### Q4：搜索失败怎么办？

自动使用内置税法常识库，并在配置文件中标记 `needs_verification: true`，提示用户可能需要人工复核。

### Q5：如何确保生成的计算器风格统一？

生成器自动遵循《统一性说明文档.md》中的规范：
- 使用统一的字体族和色板
- 遵循配置文件命名规范
- 统一数字格式化方式
- 统一错误处理方式
- 复杂税种使用分步向导模式
