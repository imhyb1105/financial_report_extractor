## 02｜📊 量变：18个税种的"复制粘贴"困境

先说量变——**18个税种，意味着什么？**

中国现行18个税种，每个都有不同的：
- 税率表（有的简单，有的复杂到让人头皮发麻）
- 计算规则（含税不含税、价内价外、累进税率...）
- 政策口径（而且会变！）

如果用传统方式——**vibe coding**，一个一个跟AI唠嗑：

```
我：帮我做一个增值税计算器...
AI：好的...
我：再做企业所得税...
AI：好的...
我：还有个税、消费税、印花税、城建税...
AI：哥，你是不是有点过分了？
```

**这不是编程，这是在当客服。** 📞

我算了一笔账：

| 项目 | Vibe Coding方式 |
|------|----------------|
| 每个税种沟通轮次 | 5-10轮 |
| 18个税种总沟通 | 90-180轮 |
| 每轮平均耗时 | 2分钟 |
| **总耗时** | **3-6小时纯聊天** |

这就是"量"的问题——**数量越多，重复劳动越可怕。**

**我的手指开始抽筋了。** 🖐️

## 03｜🔄 量变积累：从"复制"到"模式化"

做到第3个税种的时候，我发现了一个规律：

> **每个税种计算器的"零件"结构是一样的，只是内容不同。**

```
增值税计算器/
├── 增值税计算器.html      # 页面
├── script.js              # 计算逻辑
├── styles.css             # 样式
├── vat.config.json        # 税率配置
├── PRD.md                 # 需求文档
└── 增值税知识库/           # 配套知识
    ├── 0_索引.md
    ├── 1_基础知识.md
    ├── 2_税率.md
    ├── 3_规定.md
    ├── 4_计算方法.md
    └── 5_纳税调整详解.md
```

企业所得税、个人所得税、印花税...**结构完全一样！**

这时候，我的思维开始转变：

| 阶段 | 思维方式 | 行为 |
|------|----------|------|
| 做第1个 | "我要做一个计算器" | 从零开始，手忙脚乱 |
| 做第3个 | "我要复制这个模式" | 手动复制，改参数 |
| 做第5个 | "能不能让AI自动复制？" | **开始寻找工具** |

**这就是量变的第一个价值：让你发现"模式"。**

## 04｜🚀 质变发生：Skill让模式变成"系统"

做到第5个税种的时候，我找到了**Skill**。

Skill是Claude Code的一个功能，可以把"模式"封装成"系统"。

打个比方：

- **Vibe Coding** = 每次去餐厅，都要跟服务员解释："我要一份饭，不要香菜，多点辣..."
- **Skill** = 直接按"老顾客套餐"按钮，系统自动出餐

**更骚的是：Skill可以批量执行！**

![插图提示词：一个人坐在传送带旁边，传送带上自动送来一个个精美的便当盒，每个便当盒上贴着不同税种的标签（增值税、企业所得税、个人所得税等），人物表情轻松惬意，风格为现代扁平插画]

我设计的Skill核心逻辑：

```
1. 读取税种参数（名称、税率表、计算规则）
2. 按模板生成6个文件（HTML/JS/CSS/Config/PRD/知识库）
3. 保持与其他税种的样式统一
4. 自动创建目录结构
```

然后，我只需要：

> **"帮我创建企业所得税计算器，参考增值税的结构，税种参数如下..."**

AI就会自动完成全部工作。

**这就是质变：**

| 量变阶段 | 质变阶段 |
|----------|----------|
| 手动复制模板 | **模板自动执行** |
| 逐个沟通调整 | **一次指令搞定** |
| 我来控制流程 | **系统控制流程** |
| 我是"操作工" | **我是"设计师"** |

| 量变思维 | 质变思维 |

| "帮我写一段代码" | "帮我设计一套流程" |

税率变了？改个数字，push到GitHub，Vercel自动部署，**1分钟更新上线**。

## 08｜🎓 你也能触发"质变"

如果你想复刻这套操作，核心思路是：

### 第一步：积累量变——找到"模式"

问自己：
- 这个任务有没有固定的文件结构？
- 每次做的差异点在哪里？
- 哪些可以模板化？

**量变的意义，就是让你发现这些"模式"。**

### 第二步：设计质变——封装"系统"

把模式变成Skill模板：

```
角色：你是一个专业的税种计算器开发工程师
任务：根据输入的税种参数，生成完整的计算器项目
要求：
1. 文件结构：HTML/JS/CSS/Config/知识库
2. 样式统一：使用公共CSS
3. 配置分离：税率放入config.json
4. 知识库完整：6个MD文档
```

### 第三步：享受质变——批量执行

把18个税种的参数准备好，一个一个喂给Skill。

然后——

> 坐和放宽，让AI当你的外包队 🤖
