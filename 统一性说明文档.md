# 统一性说明文档（财税小工具）

版本：v1.0.0  
适用范围：增值税计算器、附加税费计算器，以及后续新增的所有财税小工具。

---

## 目标与原则
- 降低维护成本：统一技术栈、视觉规范与交互方式。
- 提升一致体验：统一数字格式、错误提示与政策可溯源性。
- 保持计算独立：各工具计算逻辑独立，但呈现与治理方式一致。

## 当前差异速览（背景）
- 技术依赖：增值税计算器使用 Bootstrap 5；附加税费计算器为原生自定义样式。
- UI结构：增值税为多标签页（一般/小规模/混合销售），含差额与明细表；附加税费为单页表单 + 结果网格。
- 字体与色板：增值税固定 `Microsoft YaHei`、蓝色系；附加税费采用系统字体族与 CSS 变量色板。
- 数值与提示：增值税结果使用千分位与 `alert()` 错误提示；附加税费统一两位小数并内联错误区。
- 配置治理：增值税税率硬编码；附加税费使用独立 `rates.config.json` 并展示版本/生效日期。
- 知识库联动：附加税费页面提供知识库链接；增值税页面未显式提供入口。

---

## 统一性总则（决策）
- 技术栈：默认采用“原生 HTML + 自定义 CSS + 原生 JS”。如确需第三方库（如 Bootstrap），须在所有工具中一致启用并统一组件用法。
- 数字格式：结果区统一“千分位 + 两位小数”。内部计算保留精度，最终输出统一格式化。
- 校验与提示：统一使用表单内联错误区（`aria-live="polite"`），避免 `alert()` 作为主要错误呈现。
- 配置治理：各工具的税率/政策口径集中在独立 `config` 文件，页面统一展示“版本/生效日期”与来源（PRD + 知识库）。

---

## 技术栈与依赖治理
- 默认不依赖外部 CDN；如确需依赖（例如图标或框架），需：
  - 在两个计算器中一致使用；
  - 明确降级方案与离线可用性；
  - 记录版本与来源于配置或 README。
- 共享基础资源建议（已落地到 `common/`）：
  - `common/styles.css`：色板、卡片、按钮、表单、栅格的统一样式。
  - `common/components/utils.js`：数字格式化、两位小数、复制与校验工具方法。
  - `common/components/version.js`：配置驱动的版本与生效日期展示。
  - `common/components/export.js`：结果区打印导出（PDF）。

## UI 与视觉统一
- 字体族统一：
  - `system-ui, -apple-system, Segoe UI, Roboto, Helvetica, Arial, PingFang SC, Microsoft YaHei, sans-serif`
- 色板统一（CSS 变量示例）：
  - `--bg: #f7f7fa`、`--card: #ffffff`、`--text: #1f2937`、`--muted: #6b7280`
  - `--primary: #2563eb`、`--primary-contrast: #ffffff`、`--danger: #dc2626`
- 组件规范：
  - 卡片：圆角 10px、轻阴影 `0 1px 2px rgba(0,0,0,0.06)`、内边距 16px。
  - 按钮：主次态（`btn-primary` / `btn-secondary`），悬浮轻微亮度变化。
  - 表单：统一焦点态（描边 + 阴影）、对齐与间距一致。
- 导航一致性：选择一种方案并贯彻：
  - 方案 A：单页 + 卡片分区（推荐，轻量一致）。
  - 方案 B：标签页（tabs）分区（如业务复杂确需分区）。

## 表单与校验统一
- 关键输入统一加 `required` 与非负校验，避免负值。
- 错误呈现：
  - 使用内联错误区：`<div class="errors" aria-live="polite"></div>`。
  - 错误文案简洁、可定位，避免弹窗中断流程。
- 可用性提示统一在结果注释区域（`note`），避免混用不同风格提示框。

## 数字与格式统一
- 统一函数语义：
  - 输入解析：`parseAmount(input)`，返回数字或 `NaN`。
  - 两位小数：`toFixed2(n)`，四舍五入至两位。
  - 结果格式：`formatMoney(n)`，千分位 + 两位小数。
- 中间计算使用原始数值；渲染阶段统一格式化。

### 生效日期规则（统一）
- 展示语义：页面顶部与页尾显示“最后生效日期”，以相关法律、法规、条例或规定的最后生效日期为准。
- 字段优先级（从高到低）：
  - `effective_until`
  - `effective_period.to` 或 `effective_period.end`
  - `effective_date`（单值即视为最后生效日期）
- 配置示例（可选生效区间）：
```json
{
  "version": "2025.11-001",
  "effective_period": { "from": "2021-04-01", "to": "2027-12-31" }
}
```
- 展示规范：若提供生效区间，顶部仍以“最后生效日期”展示；需要完整区间显示时，可在结果说明或页脚补充 `from ~ to` 文案。

## 知识库与政策联动统一
- 两计算器页面都提供知识库直达链接（基础/税率/征收规定/计算方法）。
- 页脚合规提示统一：
  - 免税/零税率/即征即退：附加税费随同税款先征后退或冲减，以“实际缴纳税额”为计征基础。
  - 进口环节：由海关一并代征，计算公式与口径一致（提示性质）。

## 文件结构与命名统一
- 入口文件命名统一：
  - 方案 A：均用 `index.html`（推荐）。
  - 方案 B：均用中文名（例如“增值税计算器.html”与“附加税费计算器.html”），但需统一风格。
- 配置文件：采用对称命名与路径：
  - `附加税费计算器/rates.config.json`
  - `增值税计算器/vat.config.json`
- 测试页：两者均保留 `test.html`，用于回归验证与示例用例。
- 公共资源目录（建议）：
```
shared/
  ├─ theme.css      # 公共色板与基础组件样式
  └─ utils.js       # 格式化与校验工具函数
```

## 结果展示统一
- 展示结构：网格列表 + 合计金额突出 + 注释说明区（统一样式类名）。
- 金额单位与文案统一，避免不同页面风格差异。
- 注释内容包含：税率/税目或档位、关键政策提示（留抵、免税阈值、即征即退）。

## 测试与预览统一
- 测试页：按工具建立典型用例与边界用例，结果对比与自我校验说明。
- 本地预览：统一以静态服务器预览。
```powershell
python -m http.server 8000
```
- 自检清单：
  - 负值与空值拦截；
  - 两位小数与千分位一致；
  - 配置读取失败的降级文案；
  - 知识库链接有效。

## 可访问性与性能
- 可访问性：统一 `label for`、可视焦点、对比度达标、`aria-*` 使用一致。
- 性能：减少外部依赖；脚本按需加载；避免阻塞渲染；移动端响应式一致。

## 优先级与落地计划
- P0（低成本高收益）：
  - 统一结果数字格式（千分位 + 两位小数）。
  - 统一错误呈现（内联 errors + aria-live）。
  - 页脚统一展示版本/生效日期与政策来源。
- P1：
  - 增值税抽配到 `vat.config.json` 并展示版本。
  - 增值税页面补齐知识库链接与页脚合规提示。
- P2：
  - 统一字体族与 CSS 变量色板；提取公共样式到 `shared/theme.css`。
- P3：
  - 增值税新增 `test.html`，与附加税费测试结构对齐。
  - 共享格式化与校验工具到 `shared/utils.js`。
- P4：
  - 统一入口命名与导航结构（单页或标签页方案二选一并贯彻）。

## 变更管理与验收
- 配置版本号与生效日期由“PRD + 知识库”驱动，页面显示与配置一致。
- 变更需包含：配置更新、页面展示更新、测试页用例更新。
- 验收标准：
  - 页面风格一致（字体、色板、组件、焦点态）。
  - 结果数字与错误呈现统一；
  - 知识库入口与合规提示完整；
  - 预览与测试通过，无控制台错误。

## 术语约定
- 差额征税：按规则对销售额进行差额扣除后计税。
- 混合销售：多税目合并销售的计税规则，分别核算/从高适用税率两模式。
- 留抵税额：当期形成的进项大于销项的余额，按政策结转或退税。

### 公共工具方法（建议）
```js
function toFixed2(n) { return (Math.round(n * 100) / 100).toFixed(2); }
function formatMoney(n) { const x = Number(n || 0); return toFixed2(x).replace(/\B(?=(\d{3})+(?!\d))/g, ','); }
function parseAmount(input) { const v = String(input ?? '').trim(); if (v === '') return NaN; const num = Number(v); return Number.isFinite(num) ? num : NaN; }
```
