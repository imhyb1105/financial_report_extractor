# SKILL.md 测试、修复、优化流程优化方案

**版本**: 1.0
**日期**: 2026-03-21
**问题背景**: 人工测试过程中发现大量问题，说明现有自动化测试流程与实际验收标准存在较大差距

---

## 一、问题根因分析

### 1.1 测试层级与实际场景脱节

| 问题类别 | 具体表现 | 根因 |
|---------|---------|-----|
| **测试用例不足** | AI 测试通过但人工测试失败 | L0-L6 层级只覆盖技术层面，未覆盖业务场景 |
| **验收标准模糊** | "80% 测试通过"不等于"功能可用" | 缺乏明确的业务验收标准 |
| **测试数据局限** | 只有标准格式 PDF | 真实用户数据多样性未覆盖 |

### 1.2 修复循环效率问题

| 问题类别 | 具体表现 | 根因 |
|---------|---------|-----|
| **迭代次数不足** | 5 次迭代后仍有问题 | 复杂问题需要更多尝试或人工介入 |
| **问题无优先级** | 小问题和大问题同等处理 | 关键问题可能被低优先级问题阻塞 |
| **修复方向偏差** | 反复修复同一类型问题 | 缺乏问题根因分析机制 |

### 1.3 人工介入不足

| 问题类别 | 具体表现 | 根因 |
|---------|---------|-----|
| **介入时机晚** | 只在迭代耗尽时暂停 | 问题累积到后期 |
| **缺乏检查点** | 无阶段性人工确认 | AI 自行判断"完成" |
| **无预发布验证** | 直接部署 | 部署后才发现问题 |

#### 问题收集与分类

```
人工测试结果收集流程:
    │
    ├─ 用户提交测试结果 (格式: HT-01: FAIL, 问题描述: xxx)
    │
    ├─ AI 自动分类问题:
    │    ├─ P0-CRITICAL: 阻塞核心功能 (如无法上传、无法提取)
    │    ├─ P1-HIGH: 核心功能异常 (如数据错误、计算错误)
    │    ├─ P2-MEDIUM: 次要功能异常 (如展示问题、格式问题)
    │    └─ P3-LOW: 体验优化 (如文案、样式)
    │
    └─ 按优先级排序，生成问题清单
```

### 3.2 优化修复循环策略

#### 3.2.1 分层修复策略

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                    分层修复策略 (优化)                                      ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                            ║
║  原策略: 所有问题平等处理，最多 5 次迭代                                     ║
║  新策略: 按优先级分层处理，每层有独立的迭代预算                               ║
║                                                                            ║
║  ┌──────────────────────────────────────────────────────────────────────┐ ║
║  │ 优先级 │ 迭代预算 │ 修复策略                    │ 超时处理          │ ║
║  ├────────┼──────────┼─────────────────────────────┼───────────────────┤ ║
║  │ P0     │ 3 次     │ 立即修复，单人深度调试       │ ⏸️ 暂停请求人工   │ ║
║  │ P1     │ 5 次     │ 优先修复，完成后验证         │ 记录待办，继续    │ ║
║  │ P2     │ 3 次     │ 批量修复，统一验证           │ 记录待办，继续    │ ║
║  │ P3     │ 2 次     │ 可选修复                     │ 记录已知问题      │ ║
║  └──────────────────────────────────────────────────────────────────────┘ ║
║                                                                            ║
║  总迭代预算: P0(3) + P1(5) + P2(3) + P3(2) = 13 次                         ║
║  (原策略: 5 次总预算)                                                       ║
║                                                                            ║
╚═══════════════════════════════════════════════════════════════════════════╝
```

#### 3.2.2 问题根因分析机制

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                    问题根因分析 (新增)                                      ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                            ║
║  在进入修复循环前，先进行根因分析:                                           ║
║                                                                            ║
║  Step 1: 问题聚类                                                          ║
║         ├─ 将相似问题归为同一类别                                           ║
║         ├─ 识别共同的根本原因                                               ║
║         └─ 生成问题关系图                                                   ║
║                                                                            ║
║  Step 2: 根因识别                                                          ║
║         ├─ 分析问题背后的根本原因                                           ║
║         ├─ 区分"症状"和"病因"                                              ║
║         └─ 确定修复的切入点                                                 ║
║                                                                            ║
║  Step 3: 修复策略生成                                                       ║
║         ├─ 针对根因制定修复方案                                             ║
║         ├─ 评估修复的影响范围                                               ║
║         └─ 预测可能引入的新问题                                             ║
║                                                                            ║
║  Example:                                                                  ║
║  ┌──────────────────────────────────────────────────────────────────────┐ ║
║  │ 问题: HT-04 数据提取失败                                               │ ║

║  └──────────────────────────────────────────────────────────────────────┘ ║
║                                                                            ║
╚═══════════════════════════════════════════════════════════════════════════╝
```
