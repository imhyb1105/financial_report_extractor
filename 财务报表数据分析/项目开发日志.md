## 项目背景

### 项目起源

**日期**: 2026-03-20

用户提出需求：需要从上市公司公告和财务报表中提取指定内容，整理到Excel并进行排版和运算，要求数据100%准确。

**核心痛点**：
- 手工操作一家公司需要十多分钟
- 财务报表格式不统一，难以标准化提取
- 数据容易抄错，需要反复核对
- 非结构化信息（风险提示、经营分析等）难以整理

**初步讨论**：
- 用户确认数据网站（如东方财富网）可以直接下载PDF
- 需要处理的是PDF年报转换为指定Excel表格并完成指标计算

## 需求讨论记录

### 第一次需求确认 (2026-03-20)

#### 用户画像确认

| 维度 | 确认内容 |
|------|---------|
| **目标用户** | 会计师、审计师、财务分析师、投资研究员、信贷审核人员等需要经常从财务报表中提取信息的各类用户 |
| **PDF来源** | 手动上传 + 自动抓取（两种都需要） |
| **提取范围** | 数据指标 + 非数据信息（经营者、投资者、债权人关注的重要信息） |
| **验证策略** | 三模型验证：两个模型分别抓取数据，第三个模型对前两个模型抓取的数据进行核对判断 |
| **数据溯源** | AI抓取的数据提供数据源，方便人工核对 |

#### 技术路径选择

**用户倾向方案**：Web应用 + 半自动化
- 用户在线上传PDF
- AI在线提取数据
- 多轮提取、比对，提供高准确率
- 用户自行输入AI大模型API Key（防止盗用）
- 支持输入多个AI模型API Key互相比对
- 同一个AI大模型多轮生成内容比对增加准确率
- 提供数据提取位置，方便后期人工快速检验

#### 关于MCP/Skill的讨论

**结论**：不使用MCP和Skill

**原因分析**：
- MCP（Model Context Protocol）适用于Claude Desktop本地工具调用，而本项目是Web应用，场景不匹配
- Skill是Claude Code的开发辅助功能，用于开发阶段，不是运行时功能

**替代方案**：
- 使用 **Function Calling** 让AI调用定义好的提取函数
- 使用 **Structured Output** 强制AI返回结构化数据
- 使用 **多模态能力** 直接理解PDF图片

### V1 MVP范围确定

**决策**：MVP阶段先做手动上传，自动抓取放V2

**MVP核心功能**：
1. PDF处理：手动上传PDF（≤50MB）
2. 模型配置：多模型API Key管理，支持7+主流AI模型
3. 数据提取：21项核心财务指标 + 5类非财务信息
4. 三模型验证：2个提取 + 1个裁决
5. 结果展示：仪表盘式布局，置信度标识
6. 导出功能：Excel导出，数据来源批注

**V2及以后**：
- 自动抓取PDF（输入股票代码）
- 批量处理
- PDF联动（点击数据跳转PDF位置）
- 多年度对比
- 账户系统 + 团队协作

### UI设计方案选择

**最终选择**：方案B（仪表盘式）

**决策理由**：
- 开发成本适中
- 功能集中，操作效率高
- 适合高频专业用户
- V2版本可升级到方案C的PDF联动功能

**备选方案**：
- 方案A（分步引导式）：学习成本低，但效率一般
- 方案C（左右对照式）：核对最方便，但开发成本高，适合V2

## 技术方案确定

### 三模型验证机制

**设计决策**：

```
┌─────────────┐         ┌─────────────┐
│   模型 A    │         │   模型 B    │
│  独立提取   │         │  独立提取   │
└──────┬──────┘         └──────┬──────┘
       │                       │
       └───────────┬───────────┘
                   ▼
          ┌─────────────────┐
          │     模型 C      │
          │   核对与裁决    │
          └─────────────────┘
```

**模型角色分配**（用户可自定义）：
- 模型A/B：负责独立提取数据
- 模型C：负责核对A、B结果并裁决

### 支持的AI模型

| 模型提供商 | 推荐角色 | 特点 |
|-----------|---------|------|
| Anthropic (Claude) | 模型A/B | 理解能力最强 |
| OpenAI (GPT-4o) | 模型A/B | 综合能力强 |
| Google (Gemini) | 模型A/B | 多模态能力突出 |
| DeepSeek | 模型C | 性价比高，适合裁决 |
| Moonshot (Kimi) | 模型A/B/C | 长文本处理优秀 |
| 智谱AI (GLM-4) | 模型A/B/C | 国产模型 |
| MiniMax | 模型C | 性价比高 |

## 功能迭代记录

### 第一轮功能补充 (2026-03-20)

#### 新增功能1：财务数据勾稽关系核对

**需求来源**：用户提出

**功能描述**：
- AI提取数据后自动进行勾稽关系核对
- 勾稽关系有误说明AI提取的数据不准确
- 自动让AI重新提取（最多3次）

**勾稽关系规则**：
| 检查项 | 公式 | 容差 |
|--------|------|------|
| 资产负债表平衡 | 总资产 = 总负债 + 净资产 | ±1% |
| 毛利润计算 | 毛利润 = 营业收入 - 营业成本 | ±1% |
| 流动资产校验 | 流动资产 ≤ 总资产 | 0% |
| 流动负债校验 | 流动负债 ≤ 总负债 | 0% |

**重试策略**：
- 第1次失败：在Prompt中强调勾稽关系
- 第2次失败：列出所有勾稽要求
- 第3次失败：标记低置信度，提示人工核对

#### 新增功能2：金额单位转换

**需求来源**：用户提出

**功能描述**：
- 财务报告可能存在亿元、万元、元等多种单位
- 用户可以选择提取数据的显示单位
- 选择后所有数据自动转换为统一单位

**支持单位**：
- 元：精确到分，如 1,234,567,890.00
- 万元：保留2位小数，如 123,456.79
- 亿元：保留2位小数，如 12.35
